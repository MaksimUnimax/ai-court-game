# Draft MVP Scope — AI Court Game

STATUS: draft_1

## Goal

Build the first rough working frame for testing court-case scenarios before adding OpenAI API, image generation, authorization or production infrastructure.

This version exists to answer one question:

Can we create interesting, solvable, click-only court scenarios and play through them correctly?

## Included in draft MVP

The draft MVP includes:

- one simple browser page;
- backend or local script support for loading a scenario document;
- scenario upload/import into the page;
- parsing scenario content into UI blocks;
- text-only case presentation;
- clickable participants;
- clickable questions;
- clickable evidence;
- clickable verdict choices;
- final explanation after verdict;
- ability to iterate on scenario format;
- document for collecting future GPT/API prompt rules.

## Not included in draft MVP

The draft MVP explicitly excludes:

- OpenAI API calls;
- automatic GPT generation;
- image generation;
- illustrations;
- authorization;
- admin/user roles;
- game currency;
- monetization;
- database persistence;
- public production deployment;
- free text player input;
- witnesses unless added later by explicit decision;
- real legal accuracy;
- multiplayer;
- scoring system.

## Scenario source

For the draft MVP, the scenario source is a document manually generated by ChatGPT.

The user uploads the scenario document to the page.

The script parses the document and distributes content into:

- case intro;
- case type;
- participants;
- participant roles;
- available questions;
- answers;
- evidence;
- unlock conditions;
- verdict options;
- correct verdict;
- explanation.

## Player input model

The player can only click.

Allowed player actions:

- open participant card;
- select predefined question;
- open evidence card;
- inspect evidence text;
- unlock new evidence or questions through clicks;
- select final verdict;
- open final explanation.

Forbidden player actions:

- typing custom questions;
- free text chat;
- manual command input;
- editing scenario during play;
- uploading images in the first draft MVP.

## Case types

The first draft MVP supports two case families:

1. Criminal case.
2. Dispute between two people.

For a criminal case:

- participants are accused person, prosecutor and defense lawyer;
- verdict options are guilty and not guilty.

For a dispute:

- participants are side A and side B;
- verdict options are side A is right and side B is right.

## First draft visual style

The first draft is text-only.

No portraits.

No evidence illustrations.

No case cover.

No generated images.

The UI may use simple blocks/cards/buttons.

The courtroom visual design can be added later.

## Main success criteria

The draft MVP is successful if:

- a scenario document can be uploaded;
- the page can parse it;
- roles and evidence are displayed in the right places;
- the player can click through the case;
- the player can reach a verdict;
- the game can show whether the verdict is correct;
- the game can show the reasoning and key evidence;
- scenario format weaknesses become visible through playtesting.
