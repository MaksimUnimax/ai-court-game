# Draft MVP Scope — AI Court Game

STATUS: draft_1

## Цель

Собрать первый рабочий каркас для тестирования сценариев судебных дел до добавления OpenAI API, генерации изображений, авторизации и продакшн-инфраструктуры.

Эта версия должна ответить на один вопрос:

Можем ли мы создавать интересные, решаемые, кликабельные судебные сценарии и проходить их правильно?

## Что входит в draft MVP

Draft MVP включает:

- одну простую браузерную страницу;
- поддержку загрузки сценарного документа через backend или локальный скрипт;
- импорт сценария на страницу;
- разбор содержимого сценария в UI-блоки;
- текстовое представление дела;
- кликабельных участников;
- кликабельные вопросы;
- кликабельные доказательства;
- кликабельные варианты вердикта;
- финальное объяснение после вердикта;
- возможность итеративно улучшать формат сценария;
- документ для сбора будущих правил prompt'ов GPT/API.

## Что не входит в draft MVP

Draft MVP явно исключает:

- вызовы OpenAI API;
- автоматическую генерацию GPT;
- генерацию изображений;
- иллюстрации;
- авторизацию;
- admin/user роли;
- игровую валюту;
- монетизацию;
- постоянное хранение в базе данных;
- публичный production deployment;
- свободный текстовый ввод игрока;
- свидетелей, если они не добавлены позже отдельным решением;
- реальную юридическую точность;
- multiplayer;
- систему очков.

## Источник сценария

Для draft MVP источник сценария — документ, вручную сгенерированный ChatGPT.

Пользователь загружает сценарный документ на страницу.

Скрипт разбирает документ и распределяет содержимое по:

- краткому описанию дела;
- типу дела;
- участникам;
- ролям участников;
- доступным вопросам;
- ответам;
- доказательствам;
- условиям разблокировки;
- вариантам вердикта;
- правильному вердикту;
- объяснению.

## Модель ввода игрока

Игрок может только кликать.

Разрешённые действия игрока:

- открыть карточку участника;
- выбрать заранее заданный вопрос;
- открыть карточку доказательства;
- изучить текст доказательства;
- разблокировать новые доказательства или вопросы кликами;
- выбрать финальный вердикт;
- открыть финальное объяснение.

Запрещённые действия игрока:

- вводить собственные вопросы;
- использовать свободный текстовый чат;
- вводить команды вручную;
- редактировать сценарий во время игры;
- загружать изображения в первом draft MVP.

## Типы дел

Первый draft MVP поддерживает две семьи дел:

1. Уголовное дело.
2. Спор между двумя людьми.

Для уголовного дела:

- участники — обвиняемый, прокурор и адвокат защиты;
- варианты вердикта — guilty и not guilty.

Для спора:

- участники — side A и side B;
- варианты вердикта — side A is right и side B is right.

## Визуальный стиль первой версии

Первая версия текстовая.

Без портретов.

Без иллюстраций доказательств.

Без обложки дела.

Без сгенерированных изображений.

UI может использовать простые блоки, карточки и кнопки.

Визуальный стиль судебного зала можно добавить позже.

## Основные критерии успеха

Draft MVP считается успешным, если:

- сценарный документ можно загрузить;
- страница может его разобрать;
- роли и доказательства отображаются в нужных местах;
- игрок может проходить дело кликами;
- игрок может дойти до вердикта;
- игра может показать, был ли вердикт правильным;
- игра может показать рассуждение и ключевые доказательства;
- слабые места формата сценария становятся видны при playtesting.
